对 HTTP 的一些了解

发表于 2016-06-07 | 分类于 HTTP

摘要:
HTTP 是一个属于应用层的面向对象的协议，由于其简捷、快速的方式，适用于分布式超媒体信息系统。

对HTTP协议的一些了解

报文格式

HTTP报文有请求报文和响应报文，他们都是由三个部分组成，分别为头行、头部和主体。

请求报文格式

1、头行有三个部分，分别为HTTP的方法、地址、HTTP的版本；例如GET xxx.xxx.com HTTP/1.1

2、头部：

3、主体： 由于是GET方法，所以请求体为空

响应式报文格式

1、头行有三个部分，分别为HTTP协议的版本、HTTP状态码、HTTP状态码的描述；例如：HTTP/1.1 200 OK

2、头部：

3、主体：由于请求的是一个页面，所以主体包含了整个html结构。

常用的HTTP方法

1、GET：从服务器获取一份文档 不包含主体 //最常用的方法
2、POST：向服务器发送需要处理的数据 包含主体
3、PUT：将请求的主体部分存储在服务器上 包含主体
4、DELETE： 从服务器上删除一份文档 不包含主体
5、HEAD：只从服务器获取文档的首部 不包含主体
6、TRACE：对可能经过代理服务器传送到服务器上去的报文进行追踪 不包含主体
7、OPTIONS：决定可以在服务器上执行哪些方法 不包含主体

HTTP的版本

1、HTTP/0.9 1991年，HTTP的原型，有很多设计缺陷
2、HTTP/1.0 取代HTTP/0.9，成为第一个广泛应用的版本
3、HTTP/1.0+ 添加持久的keep-alive连接、虚拟主机支持，以及代理连接支持，成为非官方的事实标准
4、HTTP/1.1 校正了结构性缺陷，明确语义，引入重要的性能优化措施，删除不好特性，是当前使用的版本

常见的HTTP状态码

1、200 请求成功。一般用于GET和POST方法 描述信息为：OK
2、301 资源移动。所请求资源自动到新的URL，浏览器自动跳转到新的URL 描述信息为：Moved Permanently
3、304 未修改。所请求资源未修改，浏览器读取缓存数据 描述信息为：Not Modified
4、400 请求语法错误，服务器无法理解 Bad Request
5、404 未找到资源，可以设置个性“404页面” 描述信息为：Not Found
6、500 服务器内部错误 描述信息为：Internal Server Error
